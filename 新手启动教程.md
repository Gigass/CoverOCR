# CoverOCR 新手启动教程（macOS）

本教程专为新手小白编写，将手把手教您如何在 macOS 上使用命令行启动 CoverOCR 项目。

---

## 📍 第一步：打开终端（Terminal）

### 方法一：使用 Spotlight 搜索（推荐）
1. 按键盘上的 **Command (⌘) + 空格键**
2. 在弹出的搜索框中输入：`终端` 或 `Terminal`
3. 按 **回车键** 打开终端

### 方法二：从应用程序文件夹打开
1. 打开 **访达（Finder）**
2. 点击顶部菜单栏的 **前往** → **实用工具**
3. 双击 **终端（Terminal.app）**

### 终端长什么样？
打开后，您会看到一个黑色或白色的窗口，里面有类似这样的文字：
```
gigass@MacBook-Pro ~ %
```

---

## 📂 第二步：进入项目目录

### 什么是"进入目录"？
就像在访达中打开文件夹一样，在终端中我们使用 `cd` 命令来"进入"某个文件夹。

### 具体操作步骤

在终端中输入以下命令，然后按 **回车键**：

```bash
cd /Users/gigass/DEVELOP/GitHub/CoverOCR
```

**💡 小贴士**：
- `cd` 是 "change directory"（更改目录）的缩写
- 后面跟的是您项目的完整路径
- 输入完后按回车键执行

### 如何确认已经进入正确的目录？

输入以下命令查看当前位置：
```bash
pwd
```

如果显示：
```
/Users/gigass/DEVELOP/GitHub/CoverOCR
```
说明您已经成功进入项目目录了！✅

### 查看目录里有什么文件

输入以下命令：
```bash
ls
```

您应该能看到类似这样的文件列表：
```
README.md
backend/
frontend/
start_local.sh
stop_local.sh
...
```

---

## 🚀 第三步：启动项目

### 方式一：一键启动（最简单，强烈推荐！）

直接在终端中输入：

```bash
./start_local.sh
```

然后按 **回车键**。

### 如果遇到"权限被拒绝"错误

如果看到类似 `Permission denied` 的错误，需要先给脚本添加执行权限：

```bash
chmod +x start_local.sh
```

然后再次运行：
```bash
./start_local.sh
```

### 启动过程中会发生什么？

脚本会自动帮您完成以下工作：

1. ✅ **创建 Python 虚拟环境**（大约 10-30 秒）
2. ✅ **安装后端依赖**（第一次运行需要 2-5 分钟）
3. ✅ **安装前端依赖**（第一次运行需要 1-3 分钟）
4. ✅ **下载 AI 模型文件**（约 200MB，需要 3-10 分钟）
5. ✅ **启动后端服务器**（运行在 http://localhost:8000）
6. ✅ **启动前端服务器**（运行在 http://localhost:5173）

**⏰ 首次启动预计总时间：5-15 分钟**（取决于网络速度）

### 启动成功的标志

当您看到类似这样的输出时，说明启动成功了：

```
✅ Backend is running at http://localhost:8000
✅ Frontend is running at http://localhost:5173

🎉 CoverOCR is ready!
   Frontend: http://localhost:5173
   Backend API: http://localhost:8000/docs
```

## 🌐 第四步：访问应用

### 打开浏览器访问前端界面

1. 打开您的浏览器（Safari、Chrome、Firefox 都可以）
2. 在地址栏输入：`http://localhost:5173`
3. 按回车键

您应该能看到 CoverOCR 的上传界面了！🎉

### 查看后端 API 文档（可选）

在浏览器中访问：`http://localhost:8000/docs`

这里可以看到所有可用的 API 接口。

---

## 🛑 第五步：停止项目

### 方法一：使用快捷键

在运行项目的终端窗口中：
- 按 **Control (⌃) + C**

### 方法二：使用停止脚本

打开一个新的终端窗口，进入项目目录后运行：

```bash
cd /Users/gigass/DEVELOP/GitHub/CoverOCR
./stop_local.sh
```

---

## 🔄 方式二：手动启动（了解原理）

如果您想了解启动过程的细节，可以手动分步启动：

### 1. 启动后端服务

```bash
# 进入项目目录
cd /Users/gigass/DEVELOP/GitHub/CoverOCR

# 创建并激活 Python 虚拟环境
python3 -m venv .venv
source .venv/bin/activate

# 安装后端依赖
pip install -r backend/requirements.txt

# 进入后端目录
cd backend

# 启动后端服务
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 2. 启动前端服务（需要新开一个终端窗口）

**重要**：不要关闭运行后端的终端窗口！

打开一个新的终端窗口，然后执行：

```bash
# 进入项目目录
cd /Users/gigass/DEVELOP/GitHub/CoverOCR

# 进入前端目录
cd frontend

# 安装前端依赖
npm install

# 启动前端开发服务器
npm run dev
```

## ❓ 常见问题解答

### Q1: 命令行（Terminal）在哪里？
**A**: 按 `Command + 空格`，输入 `Terminal` 或 `终端`，按回车即可打开。

### Q2: 如何进入项目目录？
**A**: 在终端中输入 `cd /Users/gigass/DEVELOP/GitHub/CoverOCR` 然后按回车。

### Q3: 如何知道自己在哪个目录？
**A**: 输入 `pwd` 命令查看当前所在目录的完整路径。

### Q4: 启动失败怎么办？
**A**: 
- 确认您已经安装了 Python 3.9+ 和 Node.js 16+
- 检查终端中的错误信息
- 尝试重新运行启动脚本

### Q5: 如何检查 Python 和 Node.js 是否已安装？
**A**: 在终端中分别运行：
```bash
python3 --version
node --version
```

### Q6: 第二次启动还需要等很久吗？
**A**: 不需要！第二次启动只需要 10-30 秒，因为依赖和模型已经下载好了。

### Q7: 如何完全关闭项目？
**A**: 在运行项目的终端窗口按 `Control + C`，或运行 `./stop_local.sh`。

### Q8: 可以同时打开多个终端窗口吗？
**A**: 可以！按 `Command + T` 可以在终端中新建标签页。
